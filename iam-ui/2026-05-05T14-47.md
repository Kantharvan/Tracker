- Investigated unintended auto-updates to Obsidian summary in iam-ui project
- Found three hooks triggering saves: `UserPromptSubmit` (on `/clear`), `PostCompact` (on auto-compact), and `Stop` (after every response)
- Root cause: `Stop` hook was running `obsidian-session-note.sh` after each Claude response, causing unwanted updates
- Removed the `Stop` hook from settings.json to restrict Obsidian saves to `/clear` command only
- Fixed trailing comma in JSON after `PostCompact` block
- `PostCompact` hook remains active for auto-compact scenarios; user to confirm if this should also be removed
