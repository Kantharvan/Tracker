- Investigated auto-update mechanism for Obsidian summary saves in iam-ui (hotfix/pin-sap-piper-1404-sdol025)
- Identified root cause: multiple hooks triggering saves beyond the intended `/clear` command — specifically `Stop` hook (runs after every Claude response) and `PostCompact` hook (runs on auto-compaction)
- Found that `Stop` hook executes `obsidian-session-note.sh` unconditionally after each response, causing unintended frequent updates
- Identified solution options: either remove/disable the `Stop` hook or add matcher conditions to restrict when saves occur
- Next step: decide on preferred approach (disable unwanted hooks vs. add matchers) and implement to limit saves to `/clear` command only
